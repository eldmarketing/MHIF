# MHIF_STYLE_GUIDE.md

## Estilo

- Escribir en español claro.
- Usar frases cortas.
- Evitar teoría innecesaria.
- Priorizar reglas, tablas y decisiones.
- Cada documento debe servir para humanos y para Claude.

## Estructura estándar de objeto

```yaml
ID:
CATEGORY:
VERSION:
AUTHOR:
STATUS:
CONFIDENCE:
RELATED_OBJECTS:
TAGS:
```

## Secciones recomendadas

1. Definición
2. Objetivo
3. Cuándo utilizarlo
4. Cuándo NO utilizarlo
5. Evidencia
6. Casos MHIF
7. Errores frecuentes
8. Referencias

## Niveles de confianza

- ★★★★★ Evidencia fuerte / datos históricos consistentes.
- ★★★★☆ Experiencia robusta.
- ★★★☆☆ Experiencia preliminar.
- ★★☆☆☆ Observación débil.
- ★☆☆☆☆ Hipótesis.
